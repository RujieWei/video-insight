# Video Insight 产品需求文档

## 1. 产品定位

**Video Insight** 是一个面向个人使用的 YouTube 英文长视频学习插件。

核心定位是：面向 AI / 科技 / 商业 / 创业等英文长视频的内容学习与知识沉淀工具。

它的核心目标是帮助用户在观看英文 YouTube 长视频时：

1.  深入理解视频内容；
    
2.  获取高质量中英字幕；
    
3.  针对字幕进行 AI 问答、笔记记录、生词积累；
    
4.  将视频中的知识沉淀为可复盘的个人学习资产。
    

---

## 2. 产品形态

### 2.1 基础形态

采用：

**Chrome 浏览器插件 + Chrome Side Panel**

用户打开 YouTube 视频页后，通过 Chrome 右侧 Side Panel 使用 Video Insight。

选择 Side Panel 的原因是：它适合在用户浏览网页时提供持续可见的侧边体验，官方 Side Panel API 支持扩展在浏览器侧边栏中展示自己的 UI。 

### 2.2 MVP 版本支持

| **项目** | **MVP 范围** |
| --- | --- |
| 视频平台 | 只支持 YouTube |
| 视频长度 | 60 分钟以内 |
| 字幕来源 | 先用 mock 字幕跑通 UI；后续接公开视频可访问字幕 |
| 视频播放 | 使用 YouTube 原页面播放器，不重新开发播放器 |
| 界面语言 | 中文 |
| 主题 | 浅色模式 |
| 登录 | Supabase Auth，邮箱 Magic Link / OTP |
| 存储 | Supabase 云端存储，支持跨设备 |
| 搜索 | 支持通过模糊匹配的方式搜索字幕、笔记<br>MVP 不做，后续迭代版本增加 |
| 导出 | 支持导出字幕、导出笔记、导出生词 为markdown文档<br>MVP 不做，后续迭代版本增加 |
| 我的资料库 | 支持查看该账号下所有解析过的视频、所有笔记、对话、生词<br>MVP 不做功能，但数据结构应在MVP就预先设计好 |
| 异步任务 | MVP 不做异步任务队列，但要有清晰解析状态和超时限制 |

---

### 2.3 Side Panel 入口状态

Side Panel 不默认展示 5 个 Tab。

只有当前视频已解析完成，或已进入可展示的部分完成状态时，才展示：

```text
总览 / 字幕 / 对话 / 笔记 / 生词
```

未进入学习态前，Side Panel 按当前页面和视频状态展示不同入口：

| **状态** | **展示内容** | **是否展示 Tab** |
| --- | --- | --- |
| 正在识别当前页面 | 识别当前视频的加载状态 | 否 |
| 当前页面不是 YouTube 视频页 | 当前视频无法解析，并说明原因 | 否 |
| 当前视频不满足解析条件 | 当前视频无法解析，并说明具体原因 | 否 |
| 当前视频满足解析条件，但尚未解析 | 展示「解析当前视频」主按钮 | 否 |
| 当前视频解析中 | 展示解析进度 | 否 |
| 当前视频已解析完成 | 默认进入「总览」Tab | 是 |
| 当前视频部分完成，且核心字幕链路可用 | 默认进入「总览」Tab，失败模块展示失败提示 | 是 |

「当前视频无法解析」是一个明确产品状态，不是普通错误弹窗。

该状态用于承载：

1.  当前页面不是 YouTube 视频页；
    
2.  无法识别 videoId；
    
3.  无法读取标题；
    
4.  无法获取 durationSeconds；
    
5.  视频时长超过 60 分钟；
    
6.  未检测到可用英文字幕；
    
7.  视频内容不是英文。
    

用户未登录不属于“当前视频无法解析”，应展示登录引导。

## 3. 用户核心流程

```text
用户打开 YouTube 视频页
↓
打开 Video Insight Side Panel
↓
插件识别当前 YouTube 视频 URL / videoId / title / durationSeconds
↓
如果当前页面不是可解析的视频页，展示「当前视频无法解析」
↓
如果当前视频满足解析条件但尚未解析，展示「解析当前视频」按钮，不展示 Tab
↓
如果视频已解析过，展示历史解析数据，并默认进入「总览」Tab
↓
未解析过的情况下，点击「解析当前视频」，系统展示解析进度
↓
生成：
- 整体摘要
- 章节划分
- 自然断句后的英文字幕
- 中文翻译
↓
解析完成，默认进入「总览」Tab
↓
用户点击「字幕」Tab，进入该tab下，展示逐句字幕，字幕随视频播放进程自动滚动
↓
用户光标选中字幕，可执行：
- 问 AI
- 记笔记
- 加入生词本
- 复制原文
↓
用户可在当前视频的：
- 对话 Tab 查看问答历史
- 笔记 Tab 查看笔记
- 生词 Tab 查看生词 / 短语
```

## 视频解析流程

### 4.1 视频基础信息

插件需要识别当前 YouTube 视频：

| **字段** | **是否必须** | **说明** |
| --- | --- | --- |
| videoId | 必须 | YouTube 视频 ID |
| url | 必须 | 当前视频 URL |
| title | 必须 | 视频标题 |
| durationSeconds | 必须 | 视频时长，用于判断是否 ≤ 60 分钟 |
| channelName | 建议 | 频道名称 |
| thumbnailUrl | 建议 | 视频封面 |
| language | 建议 | 视频语言 |
| hasEnglishCaptions | 必须 | 是否有可用英文字幕 |

---

### 4.2 解析前合法性校验

系统需要在 Side Panel 识别当前页面后进行基础校验，用于决定入口状态。

用户点击「解析当前视频」后，系统再次进行解析前校验，避免页面状态变化导致错误解析。

| 顺序 | **校验项** | **规则** | **不通过提示** |
| --- | --- | --- | --- |
| 0 | 是否 YouTube 视频页 | 必须是 YouTube 视频播放页 | 当前页面不是 YouTube 视频页，Video Insight MVP 暂只支持 YouTube 视频 |
| 1 | 是否识别到 videoId | 必须能解析 videoId | 未识别到当前视频 ID，请刷新页面后重试 |
| 2 | 是否获取到标题 | 必须能获取 title | 未能读取视频标题，请刷新页面后重试 |
| 3 | 是否获取到时长 | 必须能获取 durationSeconds | 暂时无法获取当前视频时长，无法开始解析 |
| 4 | 视频时长 | 必须 ≤ 60 分钟 | 当前视频时长超过 60 分钟，MVP 暂只支持 60 分钟以内的视频解析 |
| 5 | 是否有英文字幕 | 必须能获取英文字幕 | 当前视频未检测到可用英文字幕，MVP 暂不支持无字幕视频或非英文视频解析 |
| 6 | 视频语言 | MVP 只支持英文 | 当前视频不是英文内容，MVP 暂只支持英文视频学习 |
| 7 | 用户登录 | 必须已登录 | 请先登录后再解析视频，以便跨设备同步学习记录 |
| 8 | 是否已解析过 | 如果已解析，直接读取历史结果 |  |

若校验项 0-6 不通过，Side Panel 展示「当前视频无法解析」状态，并展示对应不通过提示。

若校验项 7 不通过，Side Panel 展示登录引导，不展示「当前视频无法解析」。

## 视频解析流程

### 解析步骤

```text
获取英文字幕
重切分字幕
翻译中文字幕
生成整体摘要
生成章节
保存解析结果
```

每一步需要展示状态：

```text
待处理 / 处理中 / 已完成 / 失败
```

### 超时与上限

| **项目** | **规则** |
| --- | --- |
| 解析超时 | 每个步骤，解析时长＞ 5 分钟超时，置为失败 |
| 超时处理 | 展示“解析超时，请稍后重试或更换较短视频” |

MVP 版本不做异步任务队列，但需要后续迭代版本支持：

```text
创建解析任务
↓
后台处理
↓
用户可离开页面
↓
任务完成后读取结果
```

### 核心步骤失败：停止解析

这些步骤失败后，不继续：

| **步骤** | **失败后处理** |
| --- | --- |
| 获取英文字幕 | 停止 |
| 字幕重切分 | 停止 |
| 中文字幕翻译 | 停止 |
| 保存解析结果 | 停止或提示重试 |

原因：  
MVP 的核心体验是“自然断句后的中英字幕学习”。如果字幕链路不可用，产品主流程无法成立。

---

### 非核心步骤失败：继续解析

这些步骤失败后，可以继续：

| **步骤** | **失败后处理** |
| --- | --- |
| 生成整体摘要 | 继续 |
| 生成章节 | 继续 |

所有解析步骤完成后，在各个tab中展示对应的解析结果（注意：非核心步骤，失败了也算完成）

若存在非核心步骤失败了，则在原本展示此内容的地方展示失败提示“生成失败”

mvp版本不做重新生成，后续迭代版本支持对生成失败的模块单独重新生产

### 空状态与失败状态

Side Panel 需要区分两类“没有内容”：

| **类型** | **适用模块** | **展示规则** |
| --- | --- | --- |
| 确实没有数据 | 对话、笔记、生词 | 展示通用「暂无数据」提示和轻量骨架图 |
| 解析或生成失败 | 摘要、章节等由系统生成的内容 | 展示专门的「生成失败」提示样式 |

原则：

1.  用户主动创建的数据为空时，使用「暂无数据」。
    
2.  系统生成任务失败时，不能展示「暂无数据」，必须明确展示「生成失败」。
    
3.  非核心生成步骤失败后，Tab 仍可进入；失败模块在原位置展示失败提示。
    

## 字幕tab

### 目标

把 YouTube 原始字幕处理成更适合学习的自然句块。

原始字幕的问题：

1.  YouTube 字幕经常按时间机械切分。
    
2.  一句完整话可能被拆成多条。
    
3.  自动翻译不够准确。
    
4.  不适合逐句学习、选中提问、记录笔记。
    

### 处理结果

每个字幕块应包含：

| **字段** | **说明** |
| --- | --- |
| startTime | 字幕块开始时间 |
| endTime | 字幕块结束时间 |
| englishText | 英文原文 |
| chineseText | 中文翻译 |
| orderIndex | 排序 |

### 字幕切分规则

以「语义完整」为第一原则，同时控制学习可读性。

具体规则：

1.  每个字幕块通常控制在 8–25 个英文单词。
    
2.  可以是一句完整英文句子。
    
3.  也可以是 2–3 个短句组合。
    
4.  不要把从句、因果关系、固定表达拆散。
    
5.  不要为了长度机械切断完整语义。
    
6.  如果一句话过长，可以按自然停顿拆为 2 个语义块。
    

### 时间戳规则

如果多个原字幕片段合并成一个新字幕块：

| **字段** | **取值** |
| --- | --- |
| startTime | 第一个原字幕片段的 startTime |
| endTime | 最后一个原字幕片段的 endTime |

## 总览 Tab

### 触发

该tab为默认tab

解析完成后，直接进入「总览」Tab。

已解析过的视频，复访时也直接进入「总览」Tab。

原因：用户需要先判断视频值不值得精看。

### 内容模块

总览页包含：

1.  视频内容整体摘要
    
2.  视频章节划分：包含章节标题、章节时间范围、章节核心观点
    

### 章节结构

每个章节包含：

| **字段** | **说明** |
| --- | --- |
| title | 章节标题 |
| startTime | 开始时间 |
| endTime | 结束时间 |
| summary | 章节摘要 |
| keyPoints | 核心观点列表 |

    
4.  MVP 不支持导出图片。
    

---

## 字幕 Tab

### 展示形式

字幕页采用歌词模式，跟随播放进度自动滚动，也支持用户手动滚动、点击锚定调转

视觉要求：

1.  当前播放字幕居中高亮。
    
2.  上下字幕弱化。
    
3.  英文在上，中文在下。
    
4.  中文默认展开，用户可手动折叠。
    
5.  整体简洁，不做复杂装饰。
    

### 播放联动

| **用户行为** | **系统响应** |
| --- | --- |
| 视频正常播放 | 字幕根据当前播放时间自动滚动 |
| 当前播放到某字幕块 | 该字幕块居中高亮 |
| 用户手动滚动字幕 | 字幕块跟随滚动，但视频不跳转不暂停，仍正常播放 |
| 用户点击字幕块 | YouTube 视频跳转到对应 startTime |
| 用户选中文字 | 展示操作浮层 |

### 选中字幕浮层

用户选中字幕文本后展示 4 个操作：

1.  问 AI
    
2.  记笔记
    
3.  加入生词本
    
4.  复制原文
    

---

MVP版本不用支持、但未来计划增加的功能：

1.  字幕搜索
    
2.  字幕导出为 Markdown
    

## 对话 Tab

### 触发

用户在字幕页选中内容，点击「问 AI」，跳转到该tab

用户也可以点击对话tab，进入该tab

### 行为

如果没有历史对话，展示通用「暂无数据」提示和轻量骨架图。

若通过在字幕页选中内容、点击「问 AI」跳转，需要同步把选中文字内容带入输入框。

默认预填：

```text
「{选中的字幕内容}」

请结合视频上下文解释这段内容。
```

用户可以直接发送，也可以编辑问题。

注意：{选中的字幕内容}无法编辑，UI样式可以处理为无法编辑、修改的样式。用户也可以点击×号去掉这段字幕内容。

### AI 回答上下文

AI 回答时应结合：

1.  用户选中的字幕
    
2.  选中字幕前后若干条上下文
    
3.  当前视频整体摘要
    
4.  当前视频章节信息
    
5.  当前视频内历史对话
    
6.  外部搜索结果，必要时使用
    

前端界面展示

1.  AI回复内容应流式返回、实时展示已返回内容
    
2.  对话流随生成进度实时自动上推，展示最底部内容，过程中用户可以手动滑动，手动滑动时对话流停止自动上推，但不影响回复内容的返回
    
3.  每当用户发出新query，对话流自动滚动到最底部
    
4.  当使用了外部搜索结果生成回答时，需要在回复的最后注明引用的url，url展示标题。但无需在回复中注明。
    

### Search Provider

建议做抽象层：

```text
SearchProvider
├─ TavilySearchProvider
├─ MockSearchProvider
├─ FutureBingSearchProvider
└─ FutureOtherSearchProvider
```

Tavily 提供面向 AI / RAG 场景的 Search API，并支持通过 API 执行搜索查询，适合作为 MVP 搜索服务。

## 笔记 Tab

### 触发

用户在字幕页选中内容，点击「记笔记」

用户也可以点击对话tab，进入该tab

### 交互流程

如果没有历史笔记，展示通用「暂无数据」提示和轻量骨架图。

若通过在字幕页选中内容、点击「笔记」跳转，系统切换到「笔记」Tab，并在笔记列表顶部新增一个未保存的编辑块。

### 笔记编辑块结构

```text
笔记编辑块
├─ 选中内容
│  ├─ 英文原文
│  └─ 中文翻译
├─ 操作按钮
│  └─ 用 AI 重新整理
├─ AI 整理结果区
│  └─ 仅用户点击“用 AI 重新整理”后弹出
│  └─ 支持重新生成
├─ 我的想法
│  └─ 用户可编辑输入框
└─ 操作按钮
   ├─ 取消
   └─ 保存笔记
```

### AI 重新整理规则

默认不展示 AI 整理区域。

用户点击「用 AI 重新整理」后：

1.  调用 AI。
    
2.  结合选中字幕及前后文。
    
3.  在不改变原意的情况下，把内容整理成更适合笔记沉淀的中文表达。
    
4.  「用 AI 重新整理」按钮消失
    
5.  生成内容展示在选中内容下方
    
6.  标题：AI 整理结果
    
7.  展示操作按钮：重新生成
    
8.  点击“重新生成”可基于上述规则重新生成新的AI整理结果
    

### 保存后的笔记卡片

每条笔记展示：

1.  英文原文
    
2.  中文翻译
    
3.  AI 整理内容，如果有
    
4.  用户评论
    
5.  对应视频时间区间（取选中字幕块的时间区间）
    
6.  点击视频跳转到对应位置
    

MVP版本不用支持、但未来计划增加的功能：

1.  笔记搜索
    
2.  笔记导出为 Markdown

## 生词 Tab

### 触发

用户在字幕页选中一段内容，点击「加入生词本」。

用户也可以点击对话tab，进入该tab

### 交互流程

如果没有历史生词，展示通用「暂无数据」提示和轻量骨架图。

1.  若通过在字幕页选中内容、点击「加入生词本」跳转，界面切换到生词tab，并在生词列表顶部新增生词卡。
    
2.  生词类型支持两种类型：单词、短语
    
3.  新增的生词，由AI根据用户选中的内容自动提取值得学习的单词 / 短语，并自动加入生词列表。
    
4.  如果AI判断有多个值得学习的单词 / 短语，则多个都添加，不局限1个。
    
5.  MVP 不做候选词确认流程，用户点击「加入生词本」后直接生成并加入。
    
6.  一个生词，对应1个生词卡。
    

### 每条生词展示

| **字段** | **说明** |
| --- | --- |
| text | 单词 / 短语 |
| type | word / phrase |
| meaningZh | 中文含义 |
| sourceSentence | 来源的英文字幕原句 |

## 后续迭代：我的资料库

重要注意事项：MVP 不做「我的资料库」，计划后续迭代增加。但本次MVP版本在设计数据结构时应考虑到未来这部分功能，提前做好设计，避免后续迭代这部分功能后底层改动过大。

为了让你知道数据结构应该怎么提前做好设计，我下面就简单讲一下我的资料库会做什么功能：

「我的资料库」计划后续使用独立 Web 承载，不在 Side Panel 中做

### 模块1：视频库

展示用户学习过的所有视频：

1.  视频封面；
    
2.  标题；
    
3.  频道；
    
4.  时长；
    
5.  解析状态；
    
6.  笔记数量；
    
7.  生词数量；
    
8.  对话数量；
    
9.  最近学习时间；
    
10.  打开视频。
    

### 模块2：笔记库

支持：

1.  查看所有视频笔记；
    
2.  按视频分组；
    
3.  按时间排序；
    
4.  回到原视频时间点；
    
5.  后续支持搜索；
    
6.  后续支持 Markdown 导出。
    

### 模块3：生词本

支持：

1.  查看所有单词 / 短语；
    
2.  word / phrase 筛选；
    
3.  查看多个来源句；
    
4.  回到原视频时间点；
    
5.  后续支持复习模式；
    
6.  后续支持导出。
    

### 模块4：对话历史

支持：

1.  按视频查看对话历史；
    
2.  查看问答关联的字幕来源；
    
3.  回到原视频时间点；
    
4.  后续支持搜索；
    
5.  长期支持跨视频问答。
