# Video Insight Vibe Coding 行动指南

这份文档用于日常开发执行。

`docs/DEVELOPMENT_PLAN.md` 保留为总路线图；本文件负责把路线图拆成可执行、可验证的小步骤。

## 使用原则

1. 每次只做一个阶段，不跨阶段实现。
2. 每个阶段都必须能在浏览器里看到结果。
3. 每个阶段完成后先验证，再进入下一阶段。
4. Phase 0-8 使用 mock 数据和本地状态，不接真实 AI、Supabase、真实 YouTube 字幕。
5. 安装依赖、修改 `.env`、接入真实 API Key、git push、部署发布，都必须单独确认。

## 默认目录约定

后续写代码时优先使用以下目录：

```text
components/  React UI 组件
hooks/       React hooks 和界面状态逻辑
services/    Chrome API、存储、后端调用等服务
types/       TypeScript 类型定义
mock/        mock 视频、字幕、总览、对话、笔记、生词数据
utils/       通用工具函数
```

原则：

- React 组件中不要塞复杂业务逻辑。
- 核心数据结构必须有 TypeScript types。
- mock 数据和真实服务调用要隔离，方便后续替换。
- 用户可见文案默认中文。

- 明确安全边界：密钥、token、`.env` 不进代码、不进 commit。

### 不做

- 不安装依赖。
- 不接 Supabase、AI、真实字幕。

## Phase 1：Chrome Extension 最小骨架

### 目标

插件能被 Chrome 加载，Side Panel 能打开。

### 要做

- 初始化 Vite + React + TypeScript + Tailwind。
- 配置 Chrome Extension Manifest V3。
- 配置 Chrome Side Panel。
- 创建 background service worker 和 content script 占位文件。
- Side Panel 只展示中文占位界面：`Video Insight 准备就绪`。

### 不做

- 不展示 5 个 Tab。
- 不识别 YouTube 视频。
- 不做解析按钮。
- 不接任何后端或 API。

### 给 Codex 的任务描述

```text
请只执行 Phase 1：创建 Chrome Extension 最小骨架。
要求使用 Vite + React + TypeScript + Tailwind，配置 Manifest V3 和 Chrome Side Panel。
Side Panel 只显示“Video Insight 准备就绪”。
不要展示 5 个 Tab，不要识别 YouTube 视频，不要做解析按钮。
完成后运行构建验证，并告诉我如何在 Chrome 加载 dist/。
```

### 完成标准

- `npm run build` 通过。
- Chrome 能加载 `dist/`。
- Side Panel 能打开。
- Side Panel 显示 `Video Insight 准备就绪`。

### 手动验收

1. 运行构建命令。
2. 打开 `chrome://extensions/`。
3. 开启开发者模式。
4. 加载 `dist/`。
5. 打开 Side Panel，检查是否看到中文占位界面。

## Phase 2：YouTube 当前视频识别

### 目标

Side Panel 能知道用户当前是否在 YouTube 视频页，以及当前视频基础信息。

### 要做

- content script 在 YouTube 页面工作。
- 识别 `youtube.com/watch`。
- 读取 `videoId`、`url`、`title`、`durationSeconds`。
- Side Panel 展示当前识别状态。
- 用户切换 YouTube 视频时，视频信息自动刷新。

### 不做

- 不判断英文字幕。
- 不判断视频语言。
- 不做解析流程。
- 不展示学习态 Tab。

### 给 Codex 的任务描述

```text
请只执行 Phase 2：实现 YouTube 当前视频识别。
content script 需要识别 youtube.com/watch，并读取 videoId、url、title、durationSeconds。
Side Panel 展示识别状态和当前视频信息。
切换 YouTube 视频时信息要自动更新。
不要做解析流程，不要展示学习态 Tab。
```

### 完成标准

- YouTube 视频页显示标题、URL、videoId、时长。
- 非 YouTube 视频页不显示学习入口。
- 切换视频后信息更新。

### 手动验收

1. 打开一个 YouTube 视频页。
2. 打开 Side Panel，检查视频信息。
3. 切换到另一个 YouTube 视频，检查信息是否更新。
4. 打开非 YouTube 页面，检查是否没有学习入口。

## Phase 3：Side Panel 入口状态

### 目标

未解析前的产品状态正确，不提前展示 Tab。

### 要做

- 实现 5 种主状态：`identifying`、`unparseable`、`readyToParse`、`parsing`、`learning`。
- 非 YouTube 视频页展示 `当前视频无法解析`。
- 视频超过 60 分钟展示 `当前视频无法解析` 并说明原因。
- 可解析但未解析的视频只展示 `解析当前视频` 按钮。
- 点击按钮后进入 mock `parsing` 状态。

### 不做

- 不展示 5 个 Tab。
- 不做真实解析。
- 登录校验先保留 TODO。
- 英文字幕和视频语言校验先保留 TODO。

### 给 Codex 的任务描述

```text
请只执行 Phase 3：实现 Side Panel 入口状态。
未解析前不要展示 5 个 Tab。
根据当前页面和视频信息展示：识别中、当前视频无法解析、解析当前视频、解析中、学习态。
当前只实现非 YouTube、无法读取视频信息、超过 60 分钟的不可解析状态。
登录、英文字幕、视频语言校验先写 TODO，不阻塞本阶段。
```

### 完成标准

- 非视频页没有 Tab。
- 超过 60 分钟的视频没有 Tab，并展示不可解析原因。
- 可解析短视频出现 `解析当前视频` 按钮。
- 点击按钮后看到解析中状态。

### 手动验收

1. 打开非 YouTube 页面，检查不可解析状态。
2. 打开超过 60 分钟的视频，检查不可解析状态。
3. 打开 60 分钟以内视频，检查是否只有解析按钮。
4. 点击解析按钮，检查是否进入解析中。

- 默认模拟全部成功。
- 全部完成后进入学习态，默认进入总览。

### 不做

- 不调用 AI。
- 不保存数据库。
- 不获取真实字幕。

### 给 Codex 的任务描述


### 完成标准

### 手动验收

2. 点击 `解析当前视频`。
3. 观察解析步骤是否逐项完成。

### 不做

## Phase 6：字幕播放联动

### 目标

字幕 Tab 像歌词一样跟随 YouTube 播放。

### 要做

- 当前字幕尽量居中显示。
- 点击字幕块让 YouTube 视频跳转到对应 `startTime`。
- 用户手动滚动字幕时，不暂停视频、不强制跳转。

### 不做

- 不做选中文字浮层。
- 不接真实字幕。
- 不做字幕搜索。

### 给 Codex 的任务描述

```text
请只执行 Phase 6：实现字幕播放联动。
content script 监听 YouTube 播放器 currentTime，并发送给 Side Panel。

点击字幕块时，控制 YouTube 视频跳转到该字幕 startTime。
不要做选中文字浮层，不接真实字幕，不做字幕搜索。
```

### 完成标准

- 播放视频时，mock 字幕能随时间高亮。

### 手动验收

## Phase 8：本地持久化

- 复访同一视频时，已解析则直接进入学习态。
- 切换视频时，展示对应视频的独立状态。

### 不做

- 不接 Supabase。
- 不做跨设备同步。
- 不做用户系统。

### 手动验收

## Phase 10：Supabase Edge Functions + ModelProvider

### 目标

真实 AI 能生成学习内容，但前端不直接接触模型 API Key。

### 要做

- 使用 Supabase Edge Functions 承载模型调用。
- 建立 `ModelProvider` 抽象。
- 先支持一个真实模型供应商。
- 实现字幕重切分 + 翻译、总览生成、笔记整理、生词提取、生词详情。
- AI 返回 JSON 需要校验，失败时展示 `生成失败`。
- 核心字幕链路失败则停止解析，非核心模块失败则允许部分完成。

### 不做

- 不接 SearchProvider。
- 不做无字幕视频语音转写。
- 不做复杂任务队列。

### 给 Codex 的任务描述

```text
请只执行 Phase 10：通过 Supabase Edge Functions 接入真实 AI。
开始前先确认是否允许配置模型 API Key。
前端不能直接调用模型 API，API Key 只能放在 Supabase Edge Function 环境变量中。
请建立 ModelProvider 抽象，先支持一个模型供应商。
AI JSON 输出必须校验，失败时展示“生成失败”。
```

### 完成标准

- 使用 mock 原始字幕能生成真实摘要、章节、字幕翻译、生词。
- API Key 不出现在前端代码、日志、commit 中。
- 生成失败时 UI 状态正确。

### 手动验收

1. 使用 mock 原始字幕触发解析。
2. 检查真实 AI 生成的摘要、章节、字幕、生词。
3. 模拟生成失败，检查 `生成失败` 状态。

## Phase 11：真实 YouTube 字幕获取

### 目标

从 mock 字幕切换到真实公开视频字幕。

### 要做

- 通过 Edge Function 中的 `CaptionProvider` 获取 YouTube 英文字幕正文。
- 首个实现使用 Supadata `native` 模式，只获取已有英文字幕，不做 AI 转写。
- 只支持可获取英文字幕正文的视频。
- 无法获取英文字幕正文时展示 `当前视频无法解析`。
- 真实字幕进入 ModelProvider 做重切分和翻译。
- 解析结果保存到 Supabase。

### 不做

- 不做语音转写。
- 不支持非英文视频。
- 不支持超过 60 分钟视频。

### 给 Codex 的任务描述

```text
请只执行 Phase 11：接入真实 YouTube 英文字幕获取。
开始前先阅读 docs/YOUTUBE_CAPTION_FEASIBILITY.md 和 docs/TECH_ARCHITECTURE.md。
使用 Edge Function 中的 CaptionProvider 获取英文字幕正文，首个 provider 使用 Supadata native。
只支持可获取英文字幕正文的视频。
无英文字幕正文时展示“当前视频无法解析”。
不要做语音转写，不支持非英文视频，不支持超过 60 分钟视频。
```

### 完成标准

- 成功标准是拿到可用字幕正文片段，不是检测到字幕轨道或字幕字段。
- Supadata native 对指定验证视频返回 `segments.length > 0`，且至少一条字幕 `text` 非空、有有效 `startTime`。
- 成功获取英文字幕正文的 YouTube 短视频能完成真实解析。
- 无法获取英文字幕正文的视频展示明确不可解析提示。
- 解析后进入真实学习态。

### 手动验收

1. 打开一个有英文字幕的短视频。
2. 点击解析并等待完成。
3. 检查真实字幕、翻译、总览。
4. 打开无英文字幕视频，检查不可解析状态。

## Phase 12：SearchProvider + 对话增强

### 目标

问 AI 时在必要情况下补充外部搜索结果。

### 要做

- 建立 `SearchProvider` 抽象。
- 优先接一个搜索服务。
- 对话回答上下文包含：选中字幕、前后文、视频摘要、章节、历史对话、必要搜索结果。
- 使用外部搜索时，在回答末尾展示引用 URL 标题。

### 不做

- 不做全局搜索。
- 不做字幕搜索。
- 不做笔记搜索。

### 给 Codex 的任务描述

```text
请只执行 Phase 12：接入 SearchProvider 增强对话。
开始前先确认搜索服务和 API Key 配置方式。
普通字幕解释优先基于视频内容回答。
需要外部背景时再调用搜索，并在回答末尾展示引用 URL 标题。
不要做全局搜索、字幕搜索、笔记搜索。
```

### 完成标准

- 普通字幕解释可只基于视频内容回答。
- 涉及外部背景的问题能引用搜索结果。
- 引用展示在回答末尾。

### 手动验收

1. 选中普通字幕提问，检查回答是否基于视频内容。
2. 提问涉及外部背景的问题，检查是否有引用链接。
3. 检查引用是否在回答末尾。

```text
解析完成后进入总览
切换 5 个 Tab

选中字幕触发四个操作
刷新后恢复当前视频学习状态
```

## 固定产品规则

- Side Panel 主状态固定为：`identifying`、`unparseable`、`readyToParse`、`parsing`、`learning`。
- 学习态出现条件：视频解析完成，或核心字幕链路可用的部分完成。
- 当前视频数据以 `videoId` 为核心索引。
- Phase 0-8 使用 mock 和 `chrome.storage`。
- Phase 9 起迁移到 Supabase。
- 后端能力默认使用 Supabase Edge Functions。
- 生词默认 AI 自动加入，不做候选确认。
- 中文字幕默认展开。
- 用户主动创建数据为空时显示 `暂无数据`。
- 系统生成失败时显示 `生成失败`。
